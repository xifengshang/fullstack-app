# 后端快速入门

## 1. 后端在这个项目里负责什么

前端负责页面、交互和发起请求；后端负责：

1. 接收前端请求。
2. 校验请求数据。
3. 读写 MongoDB。
4. 按约定格式返回 JSON。
5. 处理异常、权限和安全问题。

一次“发布文章”的完整流程：

```text
用户填写表单
  -> React 收集 title/content/author/tags
  -> Axios POST /api/posts
  -> Vite 代理为 http://localhost:5000/posts
  -> Express 路由处理请求
  -> Mongoose 写入 MongoDB
  -> 后端返回 JSON
  -> React 根据结果提示成功或失败
```

## 2. 从前端概念映射到后端

| 前端概念 | 后端对应概念 |
| --- | --- |
| React 组件 | Controller/路由处理函数 |
| TypeScript interface | 数据模型 Schema、DTO |
| React Router | Express Router |
| Axios 请求 | Express 的 `req` |
| 接口返回值 | Express 的 `res` |
| 表单校验 | 服务端参数校验 |
| localStorage | 数据库或缓存 |
| `.env` 中的前端变量 | 服务端环境变量 |

后端不能信任前端传来的任何数据。即使前端做了必填校验，服务端仍然必须重新校验，因为接口可以被浏览器之外的工具直接调用。

## 3. HTTP 请求需要看懂的内容

一个请求主要包含：

- Method：操作类型，如 `GET`、`POST`、`PUT`、`DELETE`
- URL：资源地址，如 `/posts/123`
- Headers：附加信息，如内容类型、登录令牌
- Query：查询条件，如 `/posts?page=1&pageSize=10`
- Params：路径参数，如 `/posts/:id` 中的 `id`
- Body：提交的数据，常用于 `POST` 和 `PUT`

常见方法：

| 方法 | 含义 | 示例 |
| --- | --- | --- |
| GET | 查询 | 获取文章列表 |
| POST | 创建 | 发布文章 |
| PUT/PATCH | 更新 | 修改文章 |
| DELETE | 删除 | 删除文章 |

## 4. 常见状态码

| 状态码 | 含义 | 前端通常怎么处理 |
| --- | --- | --- |
| 200 | 请求成功 | 展示数据 |
| 201 | 创建成功 | 提示成功并跳转 |
| 400 | 参数错误 | 展示表单错误 |
| 401 | 未登录 | 跳转登录页 |
| 403 | 无权限 | 提示无权限 |
| 404 | 资源不存在 | 展示空状态或 404 |
| 409 | 数据冲突 | 提示重复操作 |
| 500 | 服务端异常 | 展示统一错误提示 |

不要只根据 HTTP 状态码判断业务结果。本项目还约定了 `success` 字段，前端应同时遵循统一响应结构。

## 5. Express 的核心概念

最小服务示例：

```js
const express = require('express');

const app = express();
app.use(express.json());

app.get('/posts', async (req, res) => {
  res.json({
    success: true,
    data: []
  });
});

app.listen(5000, () => {
  console.log('Server running at http://localhost:5000');
});
```

- `app.use(express.json())`：解析 JSON 请求体。
- `app.get(...)`：注册 GET 路由。
- `req`：请求信息。
- `res`：用于返回响应。
- `app.listen(...)`：启动 HTTP 服务。

## 6. MongoDB 和 Mongoose

MongoDB 用文档保存数据，一篇文章大致如下：

```json
{
  "_id": "665c...",
  "title": "文章标题",
  "content": "# Markdown 内容",
  "author": "张三",
  "tags": ["React", "Node.js"],
  "createdAt": "2026-06-13T08:00:00.000Z"
}
```

Mongoose Schema 类似 TypeScript interface，但它在运行时生效，可以定义字段类型、必填规则和默认值：

```js
const postSchema = new mongoose.Schema(
  {
    title: { type: String, required: true, trim: true },
    content: { type: String, required: true },
    author: { type: String, required: true, trim: true },
    tags: [{ type: String, trim: true }]
  },
  { timestamps: true }
);
```

`timestamps: true` 会自动产生 `createdAt` 和 `updatedAt`。

## 7. 必须先掌握的后端关键词

- 路由 Route：URL 与处理函数的映射。
- 控制器 Controller：处理请求并返回响应。
- 模型 Model：数据库数据结构和操作方法。
- 中间件 Middleware：在请求进入控制器前执行的公共逻辑。
- 环境变量：端口、数据库地址、密钥等不能写死的配置。
- REST API：以资源为中心设计 URL 和 HTTP 方法。
- 鉴权 Authentication：确认用户是谁。
- 授权 Authorization：确认用户能做什么。
