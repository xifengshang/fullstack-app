# 项目架构与运行流程

## 1. 当前技术栈

```text
React + TypeScript
        |
      Axios
        |
   Vite 开发代理
        |
Express HTTP 服务
        |
     Mongoose
        |
      MongoDB
```

## 2. 推荐的服务端目录

当前后端代码尚未建立，建议采用下面的轻量结构：

```text
server/
├── src/
│   ├── app.js                 # 服务入口、中间件和路由挂载
│   ├── config/
│   │   └── database.js        # MongoDB 连接
│   ├── controllers/
│   │   └── postController.js  # 文章业务处理
│   ├── middleware/
│   │   ├── errorHandler.js    # 统一异常响应
│   │   └── notFound.js        # 404 处理
│   ├── models/
│   │   └── Post.js            # 文章 Schema
│   └── routes/
│       └── postRoutes.js       # 文章路由
├── docs/
├── .env
├── .env.example
└── package.json
```

职责边界：

- Route 只声明 URL、HTTP 方法和对应 Controller。
- Controller 读取参数、调用 Model、组织响应。
- Model 定义数据结构并负责数据库操作。
- Middleware 处理日志、鉴权、404 和异常等公共逻辑。

## 3. 请求如何流转

以 `GET /posts/:id` 为例：

```text
请求进入 Express
  -> JSON/CORS 等中间件
  -> postRoutes 匹配 /posts/:id
  -> postController.getPost
  -> Post.findById(id)
  -> MongoDB 返回结果
  -> Controller 返回统一 JSON
  -> 异常交给 errorHandler
```

## 4. 本项目的代理规则

前端 Axios 配置：

```ts
baseURL: '/api'
```

Vite 配置：

```ts
proxy: {
  '/api': {
    target: 'http://localhost:5000',
    rewrite: (path) => path.replace(/^\/api/, '')
  }
}
```

所以实际路径变化如下：

| 前端代码 | 浏览器请求 | 后端收到 |
| --- | --- | --- |
| `api.get('/posts')` | `/api/posts` | `/posts` |
| `api.get('/posts/123')` | `/api/posts/123` | `/posts/123` |
| `api.post('/posts')` | `/api/posts` | `/posts` |

因此当前配置下，后端路由应挂载在 `/posts`，而不是 `/api/posts`。

生产环境通常不经过 Vite 开发代理。部署时应由 Nginx、网关或后端本身处理 `/api` 路由，并通过环境变量配置前端 API 地址。

## 5. 环境变量

建议 `.env.example`：

```dotenv
PORT=5000
MONGODB_URI=mongodb://127.0.0.1:27017/blog
NODE_ENV=development
```

注意：

- `.env` 不应提交到 Git。
- `.env.example` 可以提交，但不能包含真实密码。
- 前端变量会暴露给浏览器，数据库密码和服务端密钥只能放在后端。

## 6. 启动顺序

开发环境需要：

1. MongoDB 正常运行。
2. 后端连接 MongoDB，并监听 `5000`。
3. 前端 Vite 监听 `3000`。
4. 浏览器访问 `http://localhost:3000`。

后端启动失败时优先检查：

- `MONGODB_URI` 是否正确。
- MongoDB 是否已经启动。
- `5000` 端口是否被占用。
- `.env` 是否位于 server 根目录。
- server 是否已配置 `dev` 脚本。

## 7. 推荐实现顺序

1. 建立 Express 入口和 `/health` 健康检查。
2. 配置 MongoDB 连接。
3. 创建 Post Schema。
4. 实现文章列表、详情和创建接口。
5. 增加统一 404 和异常处理。
6. 用 Postman/curl 独立测试后端。
7. 再接入 React 页面。
8. 最后补分页、登录、编辑和删除功能。
